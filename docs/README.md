# gnu.in-shell-docs Documentation Planning

This folder contains working-canon planning documents for turning the current
showcase into a complete public documentation corpus.

## Documents

- [Documentation Architecture And Set Taxonomy](documentation-architecture-set-taxonomy.md)
  - Defines the public documentation model, reader paths, set-theory taxonomy,
    canonicality rules, evidence model, page contracts, and next required pages.
  - Use this before redesigning the Index or adding new documentation sections.
- [Next Zip Drop Porting Protocol](zip-drop-porting-protocol.md)
  - Defines how to absorb the next archive drop without losing the current
    Index, navigation, favicon, rail, live-map, and regression gates.
  - Use this immediately after extracting a new archive into a scratch folder.

## Operating Rule

The visual system is currently strong enough to preserve. The next upgrades
should primarily increase documentation depth, canonicality labeling, evidence,
asset inventory, and communication structure.

Do not propagate a new archive or visual iteration across every page until the
Index contract and documentation taxonomy are both intact.
